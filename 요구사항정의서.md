# 요구사항 정의서: 데일리 시간 관리 앱 (Time Tracker)

## 1. 개요
본 애플리케이션은 사용자가 하루의 주요 목표를 설정하고, 시간 단위로 일정을 관리하며, 특정 활동(공부, 운동 등)의 소요 시간을 타이머로 측정하여 누적 기록을 관리할 수 있는 개인용 웹 애플리케이션이다. 모든 데이터는 저장되어야 하며, 과거 날짜를 선택해 이전 기록을 조회하고 수정할 수 있어야 한다.

## 2. 핵심 기능 요구사항 (Functional Requirements)

### [F-01] 날짜 선택 및 과거 데이터 조회
- **설명:** 사용자가 캘린더 UI를 통해 특정 날짜를 선택할 수 있다.
- **상세 동작:**
  - 초기 진입 시 기본값은 시스템의 '오늘' 날짜로 설정된다.
  - 날짜를 변경하면, 해당 날짜에 해당하는 '목표', '시간표', '누적 타이머' 데이터를 저장소에서 불러와 화면에 렌더링한다.
  - 과거 날짜의 데이터를 열람하고 수정할 수 있어야 한다.

### [F-02] 일일 목표 TOP 3 설정 및 기록
- **설명:** 하루 중 가장 중요한 3가지 목표를 작성하고 관리한다.
- **상세 동작:**
  - 3개의 텍스트 입력 필드를 나란히 제공한다.
  - 텍스트 입력 후 상태가 변경되면, 선택된 날짜의 데이터로 자동 저장된다.

### [F-03] 활동 타이머 측정 및 누적 시간 체크
- **설명:** 타이머를 작동시켜 특정 행위의 소요 시간을 측정하고, 이를 누적하여 관리한다.
- **상세 동작:**
  - **활동명 입력:** 타이머 시작 전, 측정할 행위의 이름(예: 공부, 운동 등)을 텍스트로 입력받는다.
  - 만약 활동명을 입력하지 않고, 타이머를 측정하기 시작하면, 타이머 종료시 어떠한 측정할 행위의 이름(예: 공부 운동 등)의 시간이었는지 입력창을 띄우고 기록한다. 
  - **타이머 제어:** '시작' 버튼과 '종료' 버튼을 제공한다. 시작 시 현재 시간을 기록하고, 종료 시 소요 시간을 분(minute) 단위로 계산한다.
  - **누적 기록 저장:** 측정된 소요 시간을 선택된 날짜의 동일한 활동명 누적 데이터에 합산하여 저장한다.
  - **누적 현황 표시:** 화면에 해당 날짜에 진행된 활동명과 총 누적 시간(분)을 리스트 또는 대시보드 형태로 출력한다.

### [F-04] 일일 시간표 (Timetable) 기록
- **설명:** 오전 06시부터 오후 23시까지 1시간 간격의 시간표를 제공하여 일정과 활동을 기록한다.
- **상세 동작:**
  - 06:00부터 23:00까지 1시간 단위로 고정된 행(Row)을 제공하여 일정별 활동을 기록한다. 
  - 회사 출근 일(business day)과 휴일(holiday)을 선택할 수 있는 선택박스를 제공한다. 
  - 회사 출근 일(business day)의 경우 일일 시간표 입력창을 06:00-09:00, 12:00-13:00, 18:00-23:00 로 한정한다. (나머지 시간대는 필요 없어서 칸 삭제 후 위로 올림)
  - 휴일(holiday)의 경우 일일 입력창을 06:00-23:00로 한정한다. 
  - 기본적으로 1시간 단위로 입력할 수 있으나, 사용자가 원하면 특정시간을 분할하여 30분단위로 쪼개서 입력할 수 있게 창의 UI를 늘릴 수 있게 한다. 

  ## 3. 수행한 기능의 내용의 저장 
  - 내용은 언제 어디서든 볼 수 있게 저장되어야 한다. (어떤 경로를 통해 접속해도, 저장된 데이터를 볼 수 있어야 함.)
  - 수행한 기능의 내용이 저장 되기 위해 "배포"행위가 필요하다면 가장 합리적인 방식을 통해 "배포"를 진행하여야 한다. 
  - 웹사이트에 들어가 해당 기능을 사용할 지라도, 해당 내용이 DB에 저장되어있어 언제든 불러올 수 있어야 한다. 
  - 작성한 UI가 고도화 되기 위해 streamlit을 버리고 next.js 등의 새로운 방법으로 UI가 고도화 될 수 있다면, 수정해야한다.
